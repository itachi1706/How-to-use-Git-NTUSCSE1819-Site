Brief Description

Note: Basically most of the features should follow this flow (everything on a Pull Request). This way it will allow us to do our own stuff simultaneously effectively
  1. Click the Branch button to create a new branch
  2. Enter your desired branch name (To sort the branch to folders, append / to the branch name)
  3. Click Create Branch to confirm branch creation
  4. Do your necessary code implementation/fixes/changes
    (See Pull from GitHub, Commit Code, Push to GitHub)
  5. When it is all done, head over to the repo page
  6. You should be able to see your branch name highlighted in yellow, find your branch and select Compare and Pull Request
  7. Note: If it does not show up, manually create the PR by going to the Pull Request tab and select "Create Pull Request", after which select your branch in the compare dropdown box
  8. Enter your Pull Request Title and Descriptions accordingly if needed. Also modify the settings on the left if you need to do so
  9. When you are done, create the PR by clicking the green Create Pull Request button
  10. Now we play the waiting game until a green tick appears and it says All checks have passed (Note: If it fails, your code is not compiling and needs to be fixed)
  11. If all checks have passed, either you or anyone else can go ahead and click the green Merge Pull Request button
  12. Continue clicking Confirm merge here
  13. When you are done you can switch back to the master branch (you may not delete the branch if needed)
